### 第5页：关键算法 - 估值分位计算
**历史分位算法：**
```python
# 核心逻辑
def calculate_percentile(current_price, history_prices):
    """
    计算当前价格在历史价格中的位置
    返回：0-100% (0%=历史最低, 100%=历史最高)
    """
    count_lower = sum(1 for p in history_prices if p < current_price)
    percentile = count_lower / len(history_prices) * 100
    return percentile
```

**分位解读：**
- **< 20%**：极度低估（买入信号）
- **20-40%**：低估（可考虑买入）
- **40-60%**：正常估值
- **60-80%**：高估（谨慎）
- **\> 80%**：极度高估（卖出信号）

### 第7页：关键问题解决（一）
**问题1：微信推送Session频繁失效**
- **现象**：每周需要重新扫码登录，推送中断
- **原因**：微信Bot Session有效期约30小时
- **解决方案**：
  - 创建`check_wechat_session.py`监控脚本
  - 设置定时任务每天4次检查（07/11/15/19点）
  - Session失效自动告警，及时重新登录
- **效果**：✅ 推送稳定性显著提升

**问题2：定时任务推送配置混乱**
- **现象**：多个任务delivery配置错误，推送失败
- **原因**：channel未明确指定，accountId使用旧值
- **解决方案**：
  - 删除所有旧任务（10个）
  - 重新创建6个正确配置的任务
  - 统一推送时间为09:25（用户要求）
- **效果**：✅ 所有任务配置正确，测试推送成功

### 第8页：关键问题解决（二）
**问题3：ETF覆盖率低（仅50%）**
- **现象**：系统只覆盖757只A股ETF，未覆盖跨境ETF（211只）
- **原因**：AkShare无港股/美股指数日度数据接口
- **尝试方案**：
  - ❌ 使用efinance库 → 连接失败
  - ❌ 抓取集思录数据 → PE/PB字段为空
  - ❌ 申请理杏仁API → 付费（199元/年）
- **当前方案**：
  - ✅ 聚焦A股ETF（准确度85%）
  - ✅ 标注"免费方案局限，未覆盖跨境ETF"
  - ⏳ 长期方案：积累3个月数据后接入付费API

**问题4：估值准确度验证**
- **现象**：用户反馈"无低估ETF"，与豆包推荐差异大
- **原因**：原系统仅覆盖8只宽基指数，未覆盖行业ETF
- **解决方案**：
  - 创建`index_mapping.py`（智能关键词匹配）
  - 开发`calculate_full_etf_valuation_v2.py`
  - 覆盖率从8只 → 757只（提升94倍）
- **效果**：✅ 成功推荐15只低估ETF（分位0.5%-5.5%）

- ⚠️ **价格分位 ≠ PE/PB分位**（准确度85%）
  - 原因：免费API不提供PE/PB历史数据
  - 影响：估值准确度略低
  - 解决方案：积累3个月数据后接入理杏仁API

**历史数据局限：**
- ⚠️ **历史数据仅1天**（刚启动归档任务）

  - 影响：当前分位计算基于价格，非PE/PB
  - 解决方案：持续运行3个月，积累真实PE/PB分位

**系统功能局限：**
- ⚠️ **无回测系统**（无法验证推荐效果）
- ⚠️ **无Web界面**（仅微信推送）
