# 为什么真实数据难获取 + 解决方案

## 一、根本原因

### 1. PE/PB数据的特殊性

**ETF ≠ 个股**，ETF本身没有PE/PB，只有它跟踪的指数才有估值。

```
ETF → 跟踪指数 → 指数PE/PB
```

这导致三个限制：
1. 必须知道ETF跟踪什么指数（映射关系）
2. 必须有指数的估值数据接口
3. 指数估值接口覆盖范围有限

### 2. 当前数据源的限制

| 数据源 | 覆盖范围 | 当前状态 |
|--------|----------|----------|
| 东财市场估值（RPT_VALUEMARKET） | 仅主流指数（沪深300/科创50/创业板等） | ✅ 已接入，覆盖40条 |
| 东财行业估值（RPT_VALUEINDUSTRY_DET） | 128个行业指数 | ⚠️ 接口存在，未映射 |
| 个股估值（PE/PB加权） | 所有股票型ETF | ❌ 需开发，成本高 |
| 新浪K线历史 | 所有ETF | ⚠️ 曾被封禁，需验证 |

### 3. 历史分位数据的困境

**历史分位需要历史数据**，但：
- 我们没有历史快照积累
- 临时用hash生成伪值 → 已停用
- 需要约250个交易日（1年）才能启用真实分位

## 三、你可以怎么做

### 方案A：接受现实，用可用的数据（推荐）

**立即可用，零成本**

| 筛选条件 | 可用数据量 | 说明 |
|----------|------------|------|
| PE绝对值 ≤ 20 | ~30-40条 | 真实PE数据中筛选 |
| PEG < 1 | ~10-20条 | 真实PE + mock增长估算 |
| 流动性 ≥ 100万 | ~50条 | 估算流动性，放宽阈值 |
| 综合筛选 | **10-20条** | 真实观察池 |

**操作步骤**：
```bash

python3 scripts/agent2_etf_screener.py

# 查看结果
cat output/low_valuation_candidates_latest.json | jq '.real_pool | length'
```

### 方案C：个股估值映射（高难度）

**可提升到90%+，但需要开发投入**

**原理**：
```
ETF持仓 → 个股PE/PB → 加权计算 → ETF估值
```

**步骤**：
1. 获取ETF持仓明细（东财基金持仓接口）
2. 获取个股PE/PB（东财个股估值接口）
3. 按市值加权计算

**成本**：
- 开发时间：2-3天
- 数据量：需维护持仓数据（每日更新）
- API调用：每只ETF需调用2-3次

## 六、核心结论

**为什么真实数据这么难获取？**

因为ETF估值 = 指数估值，而免费接口只覆盖少数主流指数。

**我能立即做什么？**

用现有40条真实PE数据筛选 + 放宽流动性条件，可以观察10-20条低估ETF。

**长期怎么办？**

积累历史数据（1年后分位可用），扩展指数映射（覆盖率30-50%），或接入付费数据源（90%+）。
