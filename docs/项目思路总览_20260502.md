> 整理时间：2026-05-02  
> 负责人：张翔豪

---

## 一、项目目标

搭建一个**ETF投资辅助智能体**，每日自动抓取全市场ETF行情和估值数据，识别低估ETF，生成投资参考报告并推送日报。

**不涉及**：实盘交易、买卖推荐、基金净值预测。

**核心价值**：数据驱动 + 行业估值分层 + 自动化每日推送。

## 三、数据源体系（三层覆盖）

| 数据源 | 覆盖范围 | 数据质量 | 原理 |
|--------|---------|---------|------|
| **乐咕乐股** `stock_index_pe_lg()` | ~47只宽基ETF（沪深300、中证500等） | ⭐⭐⭐⭐⭐ **真实历史分位**（20年数据） | 指数PE/PB历史分位直接映射 |
| **申万行业** `sw_index_first_info()` | ~126只行业ETF（消费、医药、军工等） | ⭐⭐⭐⭐ 官方PE/PB | 申万官方行业估值API |
| **巨潮资讯** cninfo | ~33只主题LOF | ⭐⭐⭐ 行业PE估算 | 国证二级行业PE |
| **新浪实时** `etf_zh_a_spot_em()` | 全部ETF成交额（~382只） | ⭐⭐⭐⭐ 实时 | 实时成交数据 |

**数据覆盖率**：98%+（剩余6只为纯主动LOF，无法归类到任何行业）。

## 四、关键脚本说明

### `scripts/etf_data_collector.py`
- 输入：无
- 输出：`data/etf_spot_latest.json`（382只ETF实时行情）
- 数据源：新浪ETF实时行情API
- 功能：获取ETF代码、名称、当前价、涨跌幅、成交量、成交额、日均成交额（20日）

### `scripts/fetch_real_index_valuation.py`
- 输入：无
- 输出：`data/index_valuation_real.json`（12个宽基指数真实估值）
- 数据源：乐咕乐股 `stock_index_pe_lg(symbol="沪深300")` 等
- 功能：拉取沪深300、中证500等宽基指数的历史PE/PB分位

### `scripts/sw_industry_valuation.py`（v3.0）
- 输入：无
- 输出：`data/sw_industry_valuation_latest.json`（31个申万行业官方估值）
- 数据源：`akshare.sw_index_first_info()` — 申万官方API
- 功能：一次性获取31个申万一级行业的TTM_PE、PB、股息率
- **关键发现**：akshare的 `index_component_sw()` 成分股数据有错误（银行代码里包含建材股），后来改用 `sw_index_first_info()` 官方数据，一次成功

### `scripts/etf_valuation_enricher.py`（v6.0）
- 输入：`etf_spot_latest.json` + `index_valuation_real.json` + `sw_industry_valuation_latest.json` + `cninfo_industry_latest.json`
- 输出：`data/etf_valuation_latest.json`（382只ETF含PE/分位数据）
- 核心逻辑：
  1. 宽基ETF → 查询乐咕乐股真实分位
  2. 行业ETF → 用关键词映射表匹配申万行业代码 → 查PE
  3. LOF基金 → 巨潮资讯国证行业PE兜底
  4. 纯主动基金 → PE=null（无法归类，诚实放弃）

### `scripts/agent2_etf_screener.py`
- 输入：`etf_valuation_latest.json`
- 输出：`output/low_valuation_candidates_latest.json`
- 筛选条件：
  - 策略1（严格）：PE≤20 **或** 分位≤30%，成交额≥1亿
  - 策略2（宽松）：PE≤25，成交额≥5000万
  - 策略3（极度低估）：PE≤15，成交额≥5000万

### `scripts/agent3_etf_comparator.py`
- 输入：`low_valuation_candidates_latest.json` + 前一天快照
- 输出：`output/change_report_latest.json` + `dialog_brief_latest.txt`
- 功能：对比今日vs昨日，识别信号

## 五、发现的Bug与修复

### Bug 1：申万加权PE公式错误（v2.0）
- **问题**：代码用 `Σ(PE×w) / Σw` 正确，但除以了 `len(stocks)` 而非权重和，导致PE被严重压缩
- **表现**：食品饮料PE=1.03（实际市场约25）
- **修复**：修正加权公式（v3.0改用官方API直接获取PE，无需手工计算）

### Bug 2：LOF fallback过度泛化
- **问题**：CNINFO_KW_MAP将所有含"LOF"/"定开"/"配置"等关键词的201只ETF全部映射到"主要消费"（PE=20.32），导致原油LOF、科创LOF全部错分
- **修复**：删除模糊fallback，改为精确关键词匹配，剩余175只无匹配的纯主动LOF保持PE=null

### Bug 3：分位估算逻辑颠倒
- **问题**：低PE被映射成高分位（误认为低PE=被高估）
- **修复**：高PE=高分位（PE贵→高分位；PE便宜→低分位）

### Bug 4：akshare申万成分API数据错误
- **问题**：akshare `index_component_sw(symbol="801710")`（银行）返回建材股（北新建材、海螺水泥），导致银行PE=65
- **修复**：弃用 `index_component_sw`，改用 `sw_index_first_info()` 直接获取申万官方估值

### Bug 5：筛选阈值单位错误
- **问题**：`5000e6 = 5,000,000,000`（50亿）而非预期的5000万
- **修复**：改为 `5e7 = 50,000,000`（5000万）

## 九、技术踩坑总结

1. **东财API被封禁**：MacBook Air开代理时，东方财富API返回 `RemoteDisconnected`，关闭代理后正常；数据中心IP可能被永久封锁，手机热点可绕过
2. **akshare行业成分数据错误**：`index_component_sw` 返回的成分股与标准申万不一致，不能直接用，必须用 `sw_index_first_info()` 官方接口
3. **申万行业代码和akshare返回的不匹配**：申万标准代码801710=银行，akshare的 `index_component_sw` 里801710实际是建材，正确的银行代码是801780
4. **新浪K线不覆盖LOF**：LOF的成交额需要从 `etf_zh_a_spot_em` 实时接口获取，不能用K线历史数据
5. **过滤条件要测试边界值**：容易写错 `5000e6`（50亿）而非 `5e7`（5000万）
