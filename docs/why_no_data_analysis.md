# 为什么获取不到数据 - 根本原因分析

## 一、当前状态

| 数据源 | 状态 | 返回字段 | 覆盖率 |
|--------|------|----------|--------|
| 东财ETF行情（fund_etf_spot_em） | ❌ **被封禁** | 应有PE/PB | 0% |
| 新浪ETF行情（akshare_sina） | ✅ 可用 | 只有行情，无PE/PB | 100% |
| 东财指数估值（RPT_VALUEMARKET） | ✅ 可用 | PE/PB | 10.5% |

## 二、问题根源

### 问题1：东财ETF行情接口被封禁

```
RemoteDisconnected: Remote end closed connection without response
```

**原因**：东财检测到异常请求，临时封锁IP

**影响**：无法获取ETF本身的PE/PB数据

### 问题2：新浪接口不提供PE/PB

新浪接口只返回行情数据：
- 价格、涨跌幅、成交量、成交额
- **没有PE、PB、估值数据**

### 问题3：指数估值接口覆盖有限

东财指数估值接口（RPT_VALUEMARKET）只覆盖：
- 000300 沪深300
- 000688 科创50
- 399006 创业板指
- 000001 上证指数

**只有4个指数，导致覆盖率仅10.5%**

## 四、解决方案

### 方案1：等待IP解封（最快）

东财的封锁通常是临时的（几小时到1天）

```bash
# 明天再试
python3 scripts/run_etf_full_pipeline.py
```

### 方案2：切换网络环境

- 切换WiFi网络
- 使用手机热点
- 重启路由器（更换公网IP）

### 方案3：扩大指数映射（推荐）

即使东财ETF接口恢复，也只能获取少数指数的PE/PB。

**可以做的改进**：

1. 扩大ETF→指数映射表（从4个扩展到30-50个）
2. 接入更多估值接口：
   - RPT_VALUEINDUSTRY_DET（128个行业指数）
   - RPT_VALUEPLATE_DET（板块估值）

### 方案4：使用其他数据源

| 数据源 | PE/PB覆盖 | 状态 |
|--------|-----------|------|
| Tushare Pro | 高 | 需注册 |
| 聚宽/掘金 | 高 | 付费 |
| 理杏仁 | 高 | 付费 |

## 六、结论

**为什么获取不到数据？**

因为东财ETF接口被封禁了（RemoteDisconnected），而备用的新浪接口不提供PE/PB数据。

**能做什么？**

1. 等待IP解封（最快）
2. 切换网络环境
3. 扩大指数映射提升覆盖率
4. 接入付费数据源（长期）
